create_upset

Render a data model as a Plotly figure (intersection-size bars, set-size bars, and the membership dot matrix). Returns a plotly.graph_objects.Figure ready for fig.show(), dcc.Graph(figure=fig), or static export via kaleido.

Data constructors

Build an UpSetData model from whichever shape your data is in. All mirror the familiar upsetplot conventions.

from_counts(counts, sep="&")

Pre-aggregated exclusive intersection sizes. Keys are set-name tuples or sep-joined strings ("Action&Drama"); the empty key is the "in no set" group. Values are the exclusive (distinct) sizes.

from_memberships(memberships, element_ids=None)

One collection of set names per element (an empty collection means the element is in no set). element_ids optionally names the elements.

from_contents(contents)

A mapping of {set_name: iterable_of_element_ids}. Elements listed more than once in a set are counted once.

from_indicators(indicators, element_ids=None)

A boolean indicator table (rows = elements, columns = sets) from any narwhals-supported dataframe (pandas, Polars, PyArrow, cuDF, Modin) or a plain mapping of boolean columns. A pandas-style index supplies element ids when present.

Intersection modes

The displayed combinations (and the dot matrix) are identical across modes; only the counting changes. In the two overlapping modes an element is counted in more than one subset, so the bars no longer sum to the total.

modeCountsPartitions the data?
"distinct"Elements in exactly the member sets and no others (the exclusive intersection).Yes — sizes sum to the total.
"intersect"Elements in all member sets, regardless of other memberships.No — subsets overlap.
"union"Elements in at least one member set.No — subsets overlap.

Sorting

Prefix any key with - to reverse. Ties are broken deterministically by the participating sets' display order.

ArgumentKeys
sort_by"cardinality" (largest first), "degree" (fewest sets first), "deviation" (most surprising vs. independence first), "input".
sort_sets_by"cardinality", "name" (alphabetical), "input".

Data model

All constructors normalize to one canonical model that the renderer consumes.

UpSetData(set_names, set_sizes, intersections)

Ordered set names with their total sizes, plus the exclusive intersections. Properties: total_size; method set_size(name).

UpSetIntersection(sets, size, elements=None)

One exclusive intersection: the elements in exactly sets. elements is retained for element-based constructors and None for from_counts. Property: degree (number of participating sets).
